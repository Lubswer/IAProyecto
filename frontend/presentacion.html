I. Motivación

¿Por qué elegimos este tema? Hoy en día, las empresas reciben miles de correos, chats y reseñas diarias. Es humanamente imposible leer y clasificar todo este volumen de información en tiempo real.

¿Qué problema aborda? Resuelve el cuello de botella en la atención al cliente: la falta de priorización. A menudo, quejas urgentes o problemas técnicos graves se pierden en una bandeja de entrada saturada.

¿Por qué es relevante? Permite medir la satisfacción del cliente de forma instantánea y organizar el flujo de trabajo (ej. mandar los correos de soporte técnico al área técnica, y los reclamos de facturación al área financiera).

¿Cuál será el aporte de la IA? Automatiza la lectura, "entiende" la intención del mensaje, extrae entidades clave (marcas, ciudades) usando spaCy y calcula matemáticamente qué tan segura está de su decisión.

II. Modelos implementados

  • TfidfVectorizer: Para convertir el texto humano en vectores numéricos entendibles por la máquina.
  • Multinomial Naive Bayes (MultinomialNB): Utilizado para clasificar la Categoría del mensaje. Es el estándar de oro tradicional para clasificación de textos.
  • Complement Naive Bayes (ComplementNB): Utilizado para el Sentimiento (Positivo, Neutral, Negativo). Diseñado específicamente para rendir mejor cuando los datos están desbalanceados (ej. muchas más quejas negativas que mensajes positivos).
  • spaCy (es_core_news_sm): Modelo avanzado de NLP usado para lematización y Reconocimiento de Entidades Nombradas (NER).

III. Estado del arte

El análisis de sentimientos mediante técnicas de Procesamiento de Lenguaje Natural (NLP) ha evolucionado significativamente en los últimos años. Tradicionalmente, este proceso se basaba en diccionarios de palabras (lexicon-based), los cuales tenían limitaciones severas para entender el sarcasmo, el contexto o las estructuras complejas de las oraciones.

En el contexto actual, el estado del arte se ha desplazado hacia modelos basados en aprendizaje supervisado (como Logistic Regression o Random Forest con vectores TF-IDF) y, más recientemente, hacia arquitecturas de Deep Learning basadas en Transformers.

Estas tecnologías permiten que las máquinas no solo identifiquen palabras clave, sino que comprendan la intención detrás de cada comentario. Nuestro proyecto ConnectaNet se alinea con este avance, aplicando técnicas de aprendizaje supervisado para ofrecer una clasificación eficiente y escalable en entornos de soporte al cliente, superando los métodos manuales tradicionales.

IV. Alcance del proyecto

Lo que SÍ hace:

El proyecto abarca el ciclo completo de vida del Machine Learning. Va desde el entrenamiento del modelo a partir de un CSV, el cálculo automático de métricas (Accuracy, F1-Score), hasta el despliegue del modelo en un servidor Backend (FastAPI) consumido en tiempo real por una interfaz Frontend web moderna, estética y funcional.

Lo que NO hace:

El sistema actual no guarda un historial de los análisis en una base de datos persistente (como SQL) a largo plazo, ni re-entrena a la IA automáticamente sin intervención manual.

V. Conclusiones

  • Se demostró que algoritmos clásicos y eficientes como Naive Bayes, combinados con técnicas de preprocesamiento modernas (Lematización), logran métricas de precisión altísimas (cercanas al 99% en entornos controlados).
  • Separar el código de entrenamiento del código de inferencia es vital. Guardar el modelo en archivos .pkl redujo el tiempo de respuesta a milisegundos, logrando una web reactiva.
  • La interfaz gráfica demostró que los resultados matemáticos puros (probabilidades y métricas) pueden presentarse de forma amigable e intuitiva para cualquier usuario.

VI. Recomendaciones y Trabajos Futuros

  • Migración a Deep Learning: A futuro, reemplazar Naive Bayes por un modelo de Transformers para detectar ironías o sarcasmos complejos.
  • Base de Datos y Dashboards: Conectar el backend a PostgreSQL para guardar todas las consultas reales y generar gráficos estadísticos mensuales de la satisfacción del cliente.
  • Bucle de retroalimentación (Feedback Loop): Agregar un botón en la interfaz web que permita al usuario corregir a la IA si se equivocó, guardando ese texto para el próximo re-entrenamiento del CSV.
